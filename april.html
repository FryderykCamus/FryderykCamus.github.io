Spectator Mode

Post by Jonah Smith, Apr 17, 2017.

  • I implemented, more for my own benefit a module which pits the computer against itself. This was relatively easy as I just had to add some parameters to the enemy procedures which allows userTurn to access it. Consequently, I can use spectator mode to quickly observe the success rate of the computer without having to play myself in a cumbersome way.

Naming Scheme

Post by Jonah Smith, Apr 10, 2017.

  • While cleaning my code I accidently caused a logic error which took a while to find and solve. This setback however forced me to examine my entire system and ultimately led me to improve both the internal documentation and several poorly designed methods. For example, previously when assigning each pictureBox a place in an array I was using their coordinates to determine their location in the grid which was a haphazard solution. Instead I realised I could simply use the naming scheme “picWater03”, create a substring of y and x, “0” “3” and then place it into the grid.

Quadrants

Post by Jonah Smith, Apr 03, 2017.

  • In continual search for better ways to improve the win rate of the computer I expanded the random fire module so that instead of picking a random square across the whole grid, the computer will fire an even number of shots amongst four quadrants logically cutting down the space more quickly and winning faster.
